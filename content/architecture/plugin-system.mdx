# Plugin System

CortexPrism supports plugins that extend its capabilities with new tools, UI panels, CLI commands, LLM providers, and middleware. Plugins run in three modes: ESM modules, MCP (Model Context Protocol) servers, and WebAssembly modules.

## Plugin Types

| Kind | Technology | Best For |
|------|------------|----------|
| **ESM** | JavaScript/TypeScript modules | Most plugins ‚Äî tools, UI, providers, middleware |
| **MCP** | Model Context Protocol servers (JSON-RPC) | Wrapping existing MCP servers as tools |
| **WASM** | WebAssembly modules (any language ‚Üí WASM) | Performance-critical or multi-language plugins |

## Plugin Lifecycle

```
DISCOVERED ‚Üí INSTALLED ‚Üí LOADING ‚Üí ACTIVE
                                      ‚Üì
                                  UNLOADING ‚Üí REMOVED
```

Lifecycle hooks: `onInstall` ‚Üí `onLoad` ‚Üí `onActivate` ‚Üí `onDeactivate` ‚Üí `onUnload` ‚Üí `onUninstall`

## Manifest Structure

```json
{
  "name": "my-plugin",
  "version": "1.0.0",
  "description": "An example plugin",
  "kind": "esm",
  "entryPoint": "./mod.ts",
  "runtime": "deno",
  "capabilities": ["tools", "network:fetch"],
  "tools": [
    {
      "name": "get_weather",
      "description": "Get current weather for a city",
      "params": [
        { "name": "city", "type": "string", "description": "City name", "required": true }
      ]
    }
  ]
}
```

## Extension Points

| Capability | Description |
|------------|-------------|
| `tools` | Register tools in the global agent tool registry |
| `cli:commands` | Add `cortex <name>` subcommands |
| `ui:panel` | Add a Web UI panel/tab |
| `ui:widget` | Add a dashboard widget |
| `config:schema` | Extend config schema with new settings |

| `memory:embedder` | Custom embedding provider |
| `events:listener` | Subscribe to the event bus |

| `middleware:post` | Post-execution middleware |

## Trust Levels

| Level | Sandbox | How Assigned |
|-------|---------|--------------|
| `untrusted` | Deno Worker sandbox (minimal perms) | No verification, or blocked/suspicious supply-chain result |
| `signed` | Deno Worker sandbox | Supply-chain ran but no hash match (unverified) |
| `trusted` | In-process (full declared perms) | Supply-chain verification passes |

## PluginContext API

Every lifecycle hook and tool `execute` receives a `PluginContext`:

- `pluginId: string` ‚Äî Plugin name
- `pluginDir: string` ‚Äî `~/.cortex/data/plugins/<name>/`
- `state: PluginStateStore` ‚Äî Persistent SQLite-backed key-value store
- `config: PluginConfigStore` ‚Äî Typed access to `~/.cortex/config.json plugins.<name>`
- `logger: PluginLogger` ‚Äî Scoped logger, prefixes `[plugin:<name>]`
- `host: HostApi` ‚Äî Register/unregister tools at runtime

## Pipeline Hooks

Plugins can intercept the agent's 12-stage execution pipeline by exporting `middlewarePre` and/or `middlewarePost` from their module and declaring `middleware:pre` / `middleware:post` in capabilities.

```
pre-assess ‚Üí post-assess ‚Üí pre-reason ‚Üí post-reason
‚Üí pre-llm ‚Üí post-llm
‚Üí pre-tool (middlewarePre) ‚Üí post-tool (middlewarePost)
‚Üí pre-reflect ‚Üí post-reflect
‚Üí pre-output ‚Üí post-output
```

Hooks receive a `PipelineContext` and return a `HookResult`:

- `abort?: { reason, message }` ‚Äî aborts the current turn
- `modifyInput?: string` ‚Äî replaces user input
- `modifyLLMResponse?: string` ‚Äî replaces raw LLM response
- `modifyOutput?: string` ‚Äî replaces final output
- `injectMessages?: Message[]` ‚Äî appends to conversation history
- `sideEffects?: SideEffect[]` ‚Äî log | metric | store | notify

Sync hooks time out after **5s**. Async hooks time out after **15s**. Plugin middleware hooks run at `priority: 50`.

## Event Subscriptions

Declare `events:listener` capability and list event types in the manifest `events` field. Subscribe via `globalEventBus.on(type, handler)` in `onLoad`.

Events: `session:start`, `session:end`, `tool:pre-execute`, `tool:post-execute`, `llm:pre-call`, `llm:post-call`, `agent:turn-start`, `agent:turn-end`, `config:change`, `daemon:status`

## Sandbox Mechanics

`untrusted` and `signed` plugins run in a **Deno Worker sandbox** using JSON-RPC over `postMessage`:

1. Host spawns `new Worker(entryPoint, { type: 'module', deno: { permissions: ... } })`
2. Worker must post `{ type: 'ready' }` within **30 seconds**
3. Host sends RPC: `{ id, method: 'getTools', params: {} }` ‚Üí worker returns tool definitions
4. On tool call: `{ id, method: 'executeTool', params: { toolName, args } }` ‚Üí worker returns `ToolCallResult`

### Capability ‚Üí Deno Worker Permission Mapping

| Capability | Deno Permission |
|------------|----------------|
| `fs:read`, `fs:list` | `read: true` |
| `fs:write`, `fs:edit`, `fs:delete` | `write: true` |
| `shell:run` | `run: true` |
| `network:fetch`, `net:outbound`, `net:inbound` | `net: true` |

## Permission Overrides

Admins can grant or deny capabilities per-plugin beyond manifest declarations. Overrides are stored in `plugins.db` and merged at load time:

```
effective = declared ‚àí denied + granted
```

## Plugin Namespacing

Marketplace plugins use `@author/name` scoping. Tool names from namespaced plugins follow `@author/plugin-name/tool_name`.

## Supply-Chain Verification

`verifyPluginIntegrity()` runs before every plugin install:

- **SHA-256 hash** check against known-good hashes per `package@version`
- **Blocked hash list** ‚Äî known-malicious hashes are rejected outright
- **Digital signature verification** ‚Äî optional GPG signature validation
- **Author reputation scoring** ‚Äî 0‚Äì100 score; configurable blocked/allowed author lists
- **Malware pattern scanning** ‚Äî 6 default patterns: `eval`, `child_process`, `rm -rf /`, `curl|sh`, `wget|sh`

`blockSuspicious` mode rejects suspicious plugins. `requireKnownHash` defaults to `false` for community marketplace compatibility.

## Plugin Dependency Resolution

Semver constraint satisfaction, topological sort for install ordering, circular dependency detection, and transitive dependency traversal.

## Marketplace

Plugins can be published to and discovered through the [CortexPrism Marketplace](/marketplace). The marketplace provides:

- Plugin discovery and search
- Version management
- Dependency resolution
- Download tracking
- User ratings and reviews

## CLI Management

```bash
# Install a plugin
cortex plugins install marketplace:cortexprism.io/plugins/my-plugin
cortex plugins install https://example.com/my-plugin/manifest.json
cortex plugins install ./my-plugin/manifest.json

# List, enable, disable, update, remove
cortex plugins list
cortex plugins enable my-plugin
cortex plugins disable my-plugin
cortex plugins update my-plugin
cortex plugins update --all
cortex plugins verify my-plugin
cortex plugins permissions my-plugin
cortex plugins remove my-plugin
```
